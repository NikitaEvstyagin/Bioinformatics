# Проект по анализу геномных последовательностей

Этот репозиторий содержит решения практических заданий по анализу геномных данных. Проект включает в себя реализацию алгоритмов для поиска мотивов, анализа последовательностей и построения модели машинного обучения для предсказания связывания белков с ДНК.

## Описание файлов

- **`prac1.1.py`: Анализ геномных последовательностей (Часть 1)**
  - Кодирует последовательности ДНК в числовые 4-мерные векторы.
  - Применяет операцию свертки к данным для поиска часто встречающихся `k`-меров (подпоследовательностей длины 7, где `k = 3 mod 6 + 4`).
  - Отбирает 6 наиболее часто встречающихся 7-меров.
  - Использует алгоритм Рабина-Карпа для подсчета числа вхождений этих 6 мотивов в датасете.
  - Реализует алгоритм для поиска потенциальных точек начала репликации (Ori) на основе анализа скосов (skew) нуклеотидов (AT-skew, GC-skew) в скользящем окне.

- **`prac1.2.py`: Анализ геномных последовательностей (Часть 2)**
  - Использует 6 мотивов, полученных в `prac1.1.py`.
  - Вычисляет расстояние Левенштейна от каждого из этих мотивов до всех уникальных 7-меров в датасете.
  - Для каждого из 6 мотивов строит множество мотивов, находящихся на расстоянии Левенштейна, равном 1.
  - Предполагая, что исходные мотивы являются консенсусными последовательностями для своих множеств, вычисляет значение оценочной функции (сумму совпадений в позициях консенсуса).

- **`find_kmers_lev1.py`: Поиск 7-меров с расстоянием Левенштейна 1**
  - Выбирает два 7-мера из первой строки датасета.
  - Находит все 7-меры в третьей строке датасета, которые находятся на расстоянии Левенштейна, равном 1, от выбранных мотивов.

- **`CNN.py`: Предсказание связывания ДНК с белком**
  - Решает задачу бинарной классификации: предсказывает переменную `bind` (связывается белок с данным участком ДНК или нет).
  - Использует сверточную нейронную сеть (CNN) с архитектурой 1D-Conv.
  - Входные данные кодируются с помощью one-hot encoding (A=1, T=2, G=3, C=4).
  - Оценивает модель с помощью метрик ROC-AUC и classification report.
  - Визуализирует процесс обучения (loss и accuracy).
